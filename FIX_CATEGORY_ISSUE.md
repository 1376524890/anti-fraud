# 修复"最新资讯"分类筛选问题 - 操作指南

## 问题描述
切换分类标签后，部分板块显示的文章数量少于预期（少于3篇或为空），但数据库中确认每个分类至少有3篇文章。

## 已完成的代码修复

### 1. ✅ 云函数增强调试日志 (`cloudfunctions/getArticles/index.js`)
已添加详细的调试日志，包括：
- 接收参数的详细信息（类型、长度、字节码）
- 查询条件的完整输出
- 查询结果的详细统计
- 当查询结果为0时，自动诊断并输出所有分类的统计信息
- 相似分类名称的检测

### 2. ✅ 前端分类切换优化 (`pages/index/index.js`)
已修复：
- 添加 `trim()` 去除分类名称前后空格
- 重置 `currentPage` 为 1（防止分页偏移）
- 清空 `articles` 数组
- 重置 `hasMore` 和 `totalCount`
- 添加分类未变化时的跳过逻辑

### 3. ✅ 诊断云函数已存在 (`cloudfunctions/checkArticleCategories/index.js`)
可用于检查数据库中的分类分布情况。

## 部署步骤

### 步骤 1：上传并部署云函数

在微信开发者工具中：

1. **部署 getArticles 云函数**
   - 右键点击 `cloudfunctions/getArticles` 文件夹
   - 选择 "上传并部署：云端安装依赖"
   - 等待部署完成（看到"上传成功"提示）

2. **部署 checkArticleCategories 云函数**（用于诊断）
   - 右键点击 `cloudfunctions/checkArticleCategories` 文件夹
   - 选择 "上传并部署：云端安装依赖"
   - 等待部署完成

### 步骤 2：运行诊断云函数

1. 打开微信开发者工具的"云开发"控制台
2. 进入"云函数" -> 找到 `checkArticleCategories`
3. 点击"测试"按钮，输入空对象 `{}`
4. 点击"运行测试"
5. 查看返回结果，重点关注：
   - `categoryStats`: 每个分类的文章数量
   - `missingCategories`: 缺少文章的分类
   - `unexpectedCategories`: 数据库中存在但前端不期望的分类名称
   - `articlesWithoutCategory`: 没有 category 字段的文章

### 步骤 3：根据诊断结果修复数据

#### 情况 A：文章缺少 category 字段
如果诊断显示有文章没有 `category` 字段，需要手动在云开发控制台补充：

1. 进入"云开发" -> "数据库" -> "articles" 集合
2. 找到对应的文章记录
3. 点击"编辑"，添加 `category` 字段，值为以下之一：
   - `刷单返利`
   - `虚假投资理财`
   - `虚假购物服务`
   - `冒充电商客服`
   - `虚假贷款`
   - `虚假征信`
   - `冒充领导熟人`
   - `冒充公检法`
   - `网络婚恋交友`
   - `网游虚假交易`

#### 情况 B：分类名称不匹配
如果诊断显示 `unexpectedCategories` 有值，说明数据库中的分类名称与前端不一致：

**常见问题：**
- 多余的空格（如：`"刷单返利 "` 或 `" 刷单返利"`）
- 中英文标点差异（如：`"刷单返利，"` vs `"刷单返利"`）
- 全角/半角字符混用

**修复方法：**
1. 在云开发控制台 -> 数据库 -> articles 集合
2. 使用"查询"功能，输入：`{ "category": "错误的分类名" }`
3. 批量选中这些记录
4. 点击"批量编辑"，将 `category` 字段改为正确的值

#### 情况 C：某些分类确实没有文章
如果 `missingCategories` 显示某些分类没有文章，需要：

1. 确认是否需要为这些分类添加文章
2. 或者从前端移除这些分类标签

### 步骤 4：测试修复效果

1. **清除缓存**
   - 在微信开发者工具中，点击"清缓存" -> "清除数据缓存"
   - 重新编译小程序

2. **测试分类切换**
   - 打开小程序首页
   - 依次点击每个分类标签
   - 观察是否正确显示文章列表

3. **查看云函数日志**
   - 打开"云开发控制台" -> "云函数" -> "getArticles"
   - 点击"日志"查看详细的调试信息
   - 重点关注：
     - 接收的 category 参数是否正确
     - 查询条件是否正确
     - 返回的文章数量是否符合预期
     - 如果返回0篇，查看自动诊断输出的分类统计

## 数据库权限检查

确保 `articles` 集合的权限设置为：

```json
{
  "read": true,
  "write": false
}
```

或者：

```json
{
  "read": "auth.openid != null",
  "write": false
}
```

**检查方法：**
1. 云开发控制台 -> 数据库 -> articles 集合
2. 点击"权限设置"
3. 确认"读权限"设置为"所有用户可读"或"仅登录用户可读"

## 常见问题排查

### Q1: 切换分类后显示"暂无相关资讯"
**排查步骤：**
1. 查看云函数日志，确认接收的 category 参数
2. 运行 `checkArticleCategories` 诊断云函数
3. 对比数据库中的分类名称和前端期望的分类名称
4. 检查是否有空格、标点符号等差异

### Q2: 部分分类正常，部分分类为空
**可能原因：**
1. 数据库中该分类的文章 category 字段值不匹配
2. 该分类确实没有文章
3. 权限问题导致部分文章被过滤

**解决方法：**
运行诊断云函数，查看 `categoryStats` 和 `missingCategories`

### Q3: 所有分类都显示为空
**可能原因：**
1. 云函数未正确部署
2. 数据库权限设置错误
3. 云开发环境 ID 配置错误

**解决方法：**
1. 重新部署云函数
2. 检查 `app.js` 中的环境 ID
3. 检查数据库权限设置

### Q4: 第一页正常，加载更多时出错
**可能原因：**
分页逻辑中的 `currentPage` 没有正确递增

**解决方法：**
已在代码中修复，确保重新部署后测试
