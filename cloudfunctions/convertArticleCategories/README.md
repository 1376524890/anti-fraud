# 分类转换操作指南

## 🎯 问题确认

数据库中的 category 字段使用了**数字**（"1", "2", "3"...），而前端期望的是**中文分类名称**。

### 数字与中文的对应关系

| 数字 | 中文分类名称 | 当前文章数 |
|------|-------------|-----------|
| "1" | 刷单返利 | 2篇 |
| "2" | 虚假投资理财 | 1篇 |
| "3" | 虚假购物服务 | 1篇 |
| "4" | 冒充电商客服 | 3篇 |
| "5" | 虚假贷款 | 3篇 |
| "6" | 虚假征信 | 3篇 |
| "7" | 冒充领导熟人 | 1篇 |
| "8" | 冒充公检法 | 7篇 |
| "9" | 网络婚恋交友 | 3篇 |
| "10" | 网游虚假交易 | 4篇 |

**注意：** 数据库中还有一些已经正确标注为中文的文章：
- "刷单返利" - 4篇
- "冒充领导熟人" - 6篇
- "虚假投资理财" - 3篇
- "虚假购物服务" - 3篇

转换后，这些分类的文章数会合并。

## 🚀 操作步骤

### 步骤 1：部署转换云函数

在微信开发者工具中：
1. 右键点击 `cloudfunctions/convertArticleCategories` 文件夹
2. 选择 "上传并部署：云端安装依赖"
3. 等待部署完成（看到"上传成功"提示）

### 步骤 2：试运行（预览转换结果）

1. 打开"云开发控制台"
2. 进入"云函数" → 找到 `convertArticleCategories`
3. 点击"测试"按钮
4. 输入：`{}`（空对象，默认为试运行模式）
5. 点击"运行测试"
6. **查看返回结果**，确认转换计划是否正确

**预期返回结果：**
```json
{
  "success": true,
  "mode": "dry_run",
  "data": {
    "totalConverted": 28,
    "errorCount": 0,
    "conversionLog": [
      {
        "id": "xxx",
        "title": "某篇文章",
        "from": "1",
        "to": "刷单返利",
        "status": "will_convert"
      },
      ...
    ],
    "message": "试运行完成，将转换 28 篇文章。如需正式转换，请传入 { \"dryRun\": false }"
  }
}
```

### 步骤 3：确认无误后，正式转换

1. 在同一个测试界面
2. 输入：`{ "dryRun": false }`
3. 点击"运行测试"
4. 等待转换完成

**预期返回结果：**
```json
{
  "success": true,
  "mode": "actual_conversion",
  "data": {
    "totalConverted": 28,
    "errorCount": 0,
    "message": "正式转换完成，成功转换 28 篇文章，失败 0 篇"
  }
}
```

### 步骤 4：验证转换结果

1. 再次运行 `checkArticleCategories` 云函数
2. 输入：`{}`
3. 查看返回结果

### 步骤 5：清除缓存并测试小程序

1. 微信开发者工具 → 点击"清缓存" → "清除数据缓存"
2. 重新编译小程序
3. 打开首页，依次点击每个分类标签
4. 确认所有分类都能正常显示文章

---

## ⚠️ 注意事项

1. **先试运行，再正式转换**
   - 试运行不会修改数据，可以安全预览
   - 确认无误后再执行正式转换

2. **转换是不可逆的**
   - 正式转换后，数字分类会被替换为中文
   - 建议先备份数据（如果需要）

3. **已有中文分类的文章不受影响**
   - 云函数只转换数字分类（"1"-"10"）
   - 已经是中文的文章保持不变

## 🔍 查看转换日志

转换完成后，可以在云函数日志中查看详细过程：

```
云开发控制台 → 云函数 → convertArticleCategories → 日志
```

日志会显示：
- 每个数字分类找到的文章数量
- 每篇文章的转换状态
- 转换成功/失败的统计
